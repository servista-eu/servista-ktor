**DatabaseSetup.kt** (`eu.servista.ktor.database`):
- `fun migrateDatabase(config: DatabaseConfig): HikariDataSource`
- Creates HikariConfig with jdbcUrl, username, password from config, maximumPoolSize=10
- Creates HikariDataSource
- Runs `Flyway.configure().dataSource(dataSource).locations("classpath:db/migration").cleanDisabled(true).load().migrate()`
- **CRITICAL:** Always set `cleanDisabled(true)` to prevent accidental data loss in production
- Returns the HikariDataSource (reused for jOOQ DSLContext)
- Log migration result count using kotlin-logging

**KafkaProducerSetup.kt** (`eu.servista.ktor.kafka`):
- `class ServistaKafkaProducer(config: KafkaConfig)` -- thin wrapper around `KafkaProducer`
- Constructor creates `KafkaProducer<String, ByteArray>` with:
  - `bootstrap.servers` from `config.brokers`
  - `key.serializer` = StringSerializer
  - `value.serializer` = ByteArraySerializer (EventEnvelopeSerializer handles Avro encoding upstream in EventBuilder)
  - `schema.registry.url` from `config.schemaRegistryUrl`
  - `acks = all` for durability
- `suspend fun send(topic: String, key: String, value: ByteArray): RecordMetadata` -- wraps the Java Future in a coroutine-friendly way using `suspendCancellableCoroutine` or kotlinx-coroutines-jdk8
- `fun close()` -- closes the underlying KafkaProducer
- Keep it thin -- the `EventBuilder` from servista-commons already handles envelope construction and context propagation

**KafkaConsumerSetup.kt** (`eu.servista.ktor.kafka`):
- `fun createServistaKafkaConsumer(kafkaConfig: KafkaConfig): ServistaKafkaConsumer` -- factory function that creates a pre-configured `ServistaKafkaConsumer` (from servista-commons) using HOCON config values
- Builds `ConsumerConfig` with:
  - `bootstrap.servers` from `kafkaConfig.brokers`
  - `group.id` from `kafkaConfig.consumer!!.groupId`
  - `auto.offset.reset` from `kafkaConfig.consumer!!.autoOffsetReset`
  - `schema.registry.url` from `kafkaConfig.schemaRegistryUrl`
  - `key.deserializer` = StringDeserializer
  - `value.deserializer` = configured for Avro (EventEnvelope deserialization)
  - `enable.auto.commit` = false (ServistaKafkaConsumer from servista-commons manages commits)
- **NOTE:** The `handler: EventContextHandler` and `deadLetterRouter: DeadLetterRouter` are service-specific -- they are NOT provided by this factory. Services register their own handler and DLQ router. The factory only pre-configures the Kafka consumer properties from HOCON. The Koin module registers this as a factory that services can use, or services can call `createServistaKafkaConsumer()` directly and provide their own handler/DLQ router.
- Alternative approach if ServistaKafkaConsumer constructor requires handler/DLQ: provide a `KafkaConsumerProperties` data class that bundles the pre-configured properties, and let services use it when constructing their `ServistaKafkaConsumer`. Choose whichever approach fits the servista-commons API at implementation time.

**ServistaModule.kt** (`eu.servista.ktor.di`):
- `fun servistaModule(config: ServistaConfig) = module { ... }`
- Always provides:
  - `single { HealthRegistry(BuildInfo.fromClasspath()) }`
- Conditionally provides (when `config.database != null`):
  - `single { migrateDatabase(config.database!!) }` -- HikariDataSource
  - `single { DSL.using(get<HikariDataSource>(), SQLDialect.POSTGRES) }` -- jOOQ DSLContext
  - `single { RlsSessionManager() }`
  - Register database readiness health check: `get<HealthRegistry>().registerReadiness("database") { ... }` -- executes `SELECT 1` via DSLContext, returns UP or DOWN
- Conditionally provides (when `config.kafka != null`):
  - `single { ServistaKafkaProducer(config.kafka!!) }`
- Conditionally provides (when `config.kafka?.consumer != null`):
  - `single { KafkaConsumerProperties(config.kafka!!) }` -- pre-configured consumer properties bundle for services to use when creating ServistaKafkaConsumer instances with their own EventContextHandler and DeadLetterRouter
